## 登录校验
### 出现问题：未登录情况下也能进行访问
### 解决方案：
1. 登录校验：检查用户是否登录，如果未登录，则返回登录页面。
   实现思路：
   统一拦截，在每个请求前检查是否有登录标记，如果没有登录标记，则跳转到登录页面。
2. 会话技术
    --- 会话：用户打开浏览器，访问web服务器资源，会话建立，直到有一方断开连接，会话结束。在一次会话中可以包含多次请求和响应。
    --- 会话跟踪: 服务器会记录用户的会话信息，通过会话ID来标识用户。
    --- 会话跟踪方案 1.客户端会话跟踪技术:cookie  
    2.服务器会话跟踪技术：session 
    3.JWT(Json Web Token)令牌技术
### JWT(Json Web Token)令牌技术
    
    第一部分 Header（头部）记录令牌类型，签名算法等
    第二部分 Payload（负载）携带一些自定义信息，如用户信息，过期时间等
    第三部分 Signature（签名）防止Token被篡改，确保安全性

   ```java
    //生成JWT令牌
    @Test
      // 定义密钥常量（统一使用）
    private static final String SECRET_KEY_STRING = "这里就是要设定的密钥";
    
    // 获取 SecretKey 对象
    private SecretKey getSecretKey() {
        return Keys.hmacShaKeyFor(SECRET_KEY_STRING.getBytes(StandardCharsets.UTF_8));
    }
    public void testGenJwt() {
    // 1. 准备要存储的数据（载荷）
    Map<String, Object> claims = new HashMap<>();
    claims.put("id", 1);        // 用户ID
    claims.put("name", "tom");  // 用户名
    
    // 2. 生成 JWT
    String jwt = Jwts.builder()
            .claims(claims)                           // 设置自定义数据
            .expiration(new Date(System.currentTimeMillis() + 3600 * 1000))  // 设置过期时间（1小时）
            .signWith(getSecretKey())                 // 用密钥签名
            .compact();                               // 生成最终的JWT字符串
    
    // 3. 输出 JWT
    System.out.println("生成的 JWT: " + jwt);}

    //校验JWT令牌，检验JWT的签名是否过期，是否有效，防止被篡改，并获取载荷中的数据
    @Test
    public void testParseJwt() {
    // 1. 待解析的 JWT 字符串
    String jwt = "这里粘贴新生成的 JWT";
    
    // 2. 解析 JWT
    Claims claims = Jwts.parser()
            .verifyWith(getSecretKey())        // 用相同的密钥验证签名
            .build()                           // 构建解析器
            .parseSignedClaims(jwt)            // 解析 JWT
            .getPayload();                     // 获取里面的数据
    
        // 3. 输出解析出的数据
        System.out.println("解析结果: " + claims);
        System.out.println("id: " + claims.get("id"));
        System.out.println("name: " + claims.get("name"));
        System.out.println("过期时间: " + claims.getExpiration());}
```
       
  ### Filter过滤器 
    1. 自定义过滤器，继承自 javax.servlet.Filter 接口，并重写其所有的方法
    2. 配置Filter:Filter类上加@WebFilter注解，配置拦截资源的路径。引导类上加@ServletComponentScan开启Servlet组件扫描，并在web.xml中配置Filter。
        
``` java
        @ServletComponentScan
        //注意：不要导错包，要导入jakarta.servlet.Filter
        @WebFilter(urlPatterns = "/*")
        
        @WebFilter(urlPatterns = "/*")
        public class DemoFilter implements Filter {
        @Override
        public void init(FilterConfig filterConfig) throws ServletException {
        System.out.println("init 初始化方法执行了");
    }

        @Override
        public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse, FilterChain filterChain) throws IOException, ServletException {
        System.out.println("拦截到请求");
        //放行，访问响应的资源
        filterChain.doFilter(servletRequest,servletResponse);
    }

        @Override
        public void destroy() {
        System.out.println("destory 销毁方法执行了");
    }
    }
 ```

### 拦截器Interceptor
1.定义拦截器接口，继承自org.springframework.web.servlet.**HandlerInterceptor**接口
2.实现方法：preHandle、postHandle、afterCompletion
preHandle：在请求处理之前执行，返回true表示继续执行后续的拦截器，返回false表示中断请求，直接返回响应
postHandle：在请求处理之后执行，无论请求是否成功，都会执行
afterCompletion：在请求处理之后执行，无论请求是否成功，都会执行
```java
//Spring Boot拦截器
@Component
public class LoginCheckInterceptor implements HandlerInterceptor {
    @Override
    //postHandle后置处理，目标方法执行后视图渲染之前
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, @Nullable ModelAndView modelAndView) throws Exception {
        System.out.println("postHandle...");

    }

    @Override
    //preHandle 前置处理，目标资源方法运行之前运行
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("preHandle...");
        return true;
    }

    @Override
    //视图渲染完毕后运行，最后运行
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, @Nullable Exception ex) throws Exception {
        System.out.println("afterCompletion...");

    }
}
```
3.配置拦截器：在springmvc配置文件中配置拦截器，并指定拦截的请求路径
```java
//拦截器配置类
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Autowired
    private LoginCheckInterceptor loginCheckInterceptor;
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
            registry.addInterceptor(loginCheckInterceptor).addPathPatterns("/**");
    }
}
```
